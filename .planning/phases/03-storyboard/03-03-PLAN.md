# Plan: 03-03 - Repair and Validation

**Requirement(s):** STORY-02

**Context:**
LLMs will sometimes output malformed JSON or fail schema validation. We need a robust repair pipeline using `json-repair` and a deterministic fallback.

**Instructions:**
1. Create `storyboard/repair.py`.
2. Implement `_bullet_fallback(sentences)` which generates a deterministic safe `LLMSceneItem` list (type: "bullet") from the sentences (truncate text, use first 3 words for query).
3. Implement `parse_and_validate(content: str, n_sentences: int) -> list[LLMSceneItem]`.
4. Try `json.loads` + `model_validate`. If it works and lengths match, return.
5. If it fails, try `json_repair.repair_json` + `model_validate`. If lengths match, return.
6. If all fails (or length mismatch), return `_bullet_fallback(sentences)`.
7. Create `storyboard/tests/test_repair.py` simulating clean JSON, malformed JSON, and completely broken responses to ensure the fallback always works and never raises.

**Acceptance Criteria:**
- `parse_and_validate` never raises an exception.
- Fallback path works correctly when given gibberish.
