# Plan: 03-04 - Pipeline Integration

**Requirement(s):** STORY-01, STORY-02, STORY-03

**Context:**
The orchestrator pipeline that ties schema, prompts, client, repair, and timing injection together, mutating the timeline dictionary.

**Instructions:**
1. Create `storyboard/pipeline.py`.
2. Implement `inject_timing(items, sentences)` which takes `LLMSceneItem`s and creates `TimelineScene`s by taking start/end from `sentences`.
3. Implement `storyboard_pipeline(timeline, base_url, api_key, model, output_path)` that runs the full sequence: prompt -> llm -> parse_and_validate -> inject_timing -> save.
4. Export `storyboard_pipeline` in `storyboard/__init__.py`.
5. Create `storyboard/tests/test_pipeline.py`. Use `unittest.mock.patch` to mock `call_llm` and verify that the pipeline correctly processes the timeline and writes the updated timeline with `scenes` populated.

**Acceptance Criteria:**
- Full pipeline runs correctly without actual API keys (when mocked).
- `timeline.json` schema is correctly mutated to include the `scenes` array.
- Scene boundaries strictly match sentence boundaries.
