# Plan: 06-02 - Gradio UI

**Requirement(s):** PLAT-01

**Context:**
We need a Gradio interface that accepts a script and a voice, and outputs a video. We should present this as an extension of the existing demo concepts. Instead of modifying `VibeVoice/app.py` in-place, we will create `app.py` at the root of the workspace. This root `app.py` will serve as the entry point for the full Pipeline, whereas `VibeVoice/app.py` remains focused purely on TTS. This ensures clean separation of concerns while delivering the full pipeline UI.

**Instructions:**
1. Create `app.py` in the root of the project:
   - Import `gr.Interface` from gradio.
   - Define inputs:
     - Text (the script)
     - Voice Dropdown (scan `voices/` or `VibeVoice/demo/voices/`)
     - Backend URL (for TTS Colab GPU endpoint)
   - Define outputs:
     - A status Textbox or HTML area for progress.
     - A Video component for the final rendered output.
   - Connect the inputs/outputs to the `orchestrate_video` generator.
2. The UI should run asynchronously so that the generator updates the status Textbox while the pipeline runs.
3. Handle cleanup of the intermediate files or just store them in a unique temp folder per run using `tempfile`.

**Acceptance Criteria:**
- `python app.py` launches a Gradio server successfully.
- UI has all necessary inputs and outputs.
- Running the pipeline via UI correctly updates the status log and displays the final video.
