# Plan: 06-01 - Pipeline Orchestrator

**Requirement(s):** PLAT-02

**Context:**
We have separate modules for each phase (tts, align, storyboard, render, post_process). We need an orchestrator to run them in sequence, passing the output of one step as the input to the next. The orchestrator must handle transient failures using `tenacity` retries.

**Instructions:**
1. Create `platform/orchestrator.py`:
   - Import all phase pipelines (`tts.server`, `align.align_pipeline`, `storyboard.pipeline`, `render.render_bridge`, `post_process.post_processor`).
   - Create a `PipelineOrchestrator` class or an `orchestrate_video(script: str, voice: str, backend_url: str)` generator function.
   - The generator should yield state updates (e.g., `yield "Generating audio...", None`).
   - Use `tenacity.retry` decorators around external API calls (e.g., TTS request, LLM API) to automatically retry on network/Colab disconnects.
2. Implement intermediate file saving:
   - Save intermediate artifacts (`audio.wav`, `timeline.json`, `rendered.mp4`) in a unique temporary directory or structured output folder.
3. Write `platform/tests/test_orchestrator.py`:
   - Mock all pipeline dependencies.
   - Ensure the sequence executes in the correct order.
   - Verify `yield` statements emit the correct progress states.

**Acceptance Criteria:**
- Orchestrator function runs the 5 steps in order.
- Yields progress strings correctly.
- Test suite passes.
